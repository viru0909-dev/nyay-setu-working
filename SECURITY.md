# Security Policy

We take the security of Nyay Saarthi and its users very seriously. This document outlines our security policies and how to report vulnerabilities.

<hr/>

## Supported Versions

We are committed to keeping this project secure. Currently, the following versions receive security updates:

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |
| < 1.0   | No        |

<hr/>

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please **do not open a public issue**. This helps protect our users and prevents exploitation before a patch is available.

Please report vulnerabilities by following these steps:

1. **Contact:** Reach out to the maintainers via the email address listed on their GitHub profile or via the project's official contact channel.
2. **Details:** Provide a thorough description of the vulnerability, including steps to reproduce the issue and the potential impact it could have.
3. **Timeline:** We aim to acknowledge all reports within 48 hours and will provide a fix as quickly as possible.

<hr/>

## Responsible Disclosure

We ask that you practice responsible disclosure by giving us a reasonable amount of time to address the issue before making any information public. Thank you for helping keep our community safe.
